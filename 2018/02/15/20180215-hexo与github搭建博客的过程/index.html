hexo + github 搭建博客的过程

web hexo markdown 2018/02/15

github和hexo

程序员多多少少都会对github有一定的了解，在gtihub上建自己的个人博客，省去了购买服务器和域名的过程，对于有兴趣打理博客的童鞋是个不错的入门选择。
本文不是一篇在github上搭建博客的教程，主要是记录我在此过程中遇到的一些问题，还有就是理清在此过程中接触的一些新东西之间的关系。搭建博客的过程可以看文章结尾参考文献

主要接触到的一些知识点有：

  • github Pages
  • git环境的准备及作用
  • hexo静态博客搭建工具
  • yaml语法
  • Markdown的语法

前期准备

本文所用的操作系统是win7。

  1. 下载并安装nodeJs
  2. 下载并安装git

nodeJs用于安装搭建博客框架hexo，git环境用于上传hexo生成的文件到github。

我之前一直是用的SVN，对git并不是很熟悉。但借助于hexo，我们可以暂时不用理会任何git命令，只需要借助hexo提供的各种命令就可完成博客的生成，上传。所以，只要在github上跑通了个人主页，剩下的所有工作都可以在hexo内完成。

关于github Pages

github pages主要分为两类，一类是个人主页，一类是项目主页。个人主页有且仅有一个，我们搭建博客主要是借助于个人主页。

  1. 在github上新建一个repo，然后以username.github.io的形式命名，新建完成之后，这就是你的个人主页。
  2. 进入该repo的主页，在页面找到Settings选项，在该选项的页面找到Github Pages设置项，选择一个theme。
  3. 回到该repo的主页就可看到多了两个文件，一个是_config.yml文件，一个是index.md文件。
  4. 至此就可以在浏览器中输入https://username.github.io查看在github Pages中设置的个人主页。
    接下来就可以在线下搭建自己的个人博客并上传到该repo。

关于hexo

hexo的官方文档

hexo是一种博客框架，在该框架内提供了一系列方法可以完成创建框架，新建博客，上传github等等一系列工作。

两个命令：

  1. hexo g 生成静态文件，即需要上传到github上的文件
  2. hexo d 上传静态文件到github

关于配置文件中的layout

查看hexo 执行初始化命令hexo init [folder]之后的配置文件_config.yml,很多配置项都可以理解，只是对default_layout项不是太理解，后来专门查了下，现在记录一下。
default_layout的含义我现在的理解是对框架中文章的分类。在hexo的默认值有三个：post，draft，page 。这三个值分别对应scaffolds文件夹中的三个.md文件。
通过命令hexo new [layout] <title>可以生成一篇新的文章时，其中layout是可选项，如果不选，就用配置文件中设置的默认值，title的值中如果有空格，需要讲title用引号括起来，这是YAML语法决定的。
至于layout值的效果，简单的说就是：在生成一篇新的文章时，layout的值选用哪个就会在sources文件内生成对应的文件夹，如layout的值是draft，则会在sources中生成_draft文件夹，新生成的文章就在_draft 文件夹内，当然，post是默认值，所以_post文件夹在hexo初始化的时候就存在。
当值为post的时候，执行命令hexo clean && hexo g && hexo d的时候，_post文件夹中的文件会被直接发布到博客上；
当值为draft的时候，执行命令hexo clean && hexo g && hexo d不会将对应文件夹中的值发布到博客中，draft就是草稿的意思，如果想要预览草稿的效果，可以在编译的时候加上参数hexo g --draft，如果要将草稿正式发布，需要单独执行hexo pulish draft 文件名,将该文件移动到_post文件夹中，然后执行hexo g命令重新编译后再发布。
当值为page的时候，执行命令hexo clean && hexo g && hexo d,文件也会被发布到博客中，但其所在的位置是https://username.github.io/about中。

tips：
可以设置自定义的layout值，但自定义的layout值一般都放在_post文件夹内。

YAML语法

hexo 的配置文件用了YAML语言，后缀名是.yml，YAML是专门用来写配置文件的语言，很好理解，也很方便，可以在参考文档中打开苑一峰老师的博文了解一下。

Markdown语法

hexo写博文主要用的就是markdown语法，这个可以在写文章的时候学习一下，入门还是比较简单的，参考文档中有语法说明的链接。

关于编辑器

　　现在使用过的编辑器如下：

  • CMD Markdown : 全平台通用的一个产品，还有web版，使用起来很舒服。支持MarkDown的各种高级用法；新手必备的实时预览也很给力，可以实现左右两边的同步滚动；但是普通会员不支持对本地文件的写操作，所有文件的改动被自动保存到云端，费升级到高级会员后，才可以对本地文件进行读写。PM的这波操作，我也是大写的服!!!
  • Markdown Pad : 据说是windows下最好用的MD编辑器，但我上手之后的使用体验并不如CMD Markdown好用，首先就是实时预览的左右两边没实现同步滚动，然后对一些高阶用法的支持也不是很友好（例如todo列表）。但至少没有操蛋的不能对本地文件进行读写，现在还在使用中

关于域名绑定

将username.github.io绑定到自己的域名，需要注意以下几点

  1. 在github page的repo的根目录下，添加一个文件，文件名是CNAME，没有后缀名，文件内容是你的域名，no http，no www，just domainName.com.在hexo框架下，可以将该文件放到source文件夹下，在每次hexo d的时候，同时提交到github
  2. 关于域名解析。域名必须实名认证，否者会被暂停解析。
