webpack调试

发表于 2018-02-25

说明：

webpack 版本：3.6.0
vue 版本：2.5.2

正文

开发环境下，几乎是必须要在浏览器中进行调试的，webpack对文件进行打包之后，所有的相关文件都杂糅在了一起，如果在这里进行调试，那肯定是反人类的。所以，webpack有一套机制对源文件进行了映射，我们只需要在配置文件中进行配置即可。

通过设置webpack配置文件中的devtool的值，可以选择不同的调试模式。开发环境下的设置是在/config/index.js文件中，设置dev对象中的devtool属性值，webpack的默认值是cheap-module-eval-source-map。所有devtool的值。

在生产环境中最好关闭调试选项，将bulid对象中的productionSourceMap属性设为false即可

关于cheap-module-eval-source-map 值

  • cheap 模式可以大幅提高 souremap 生成的效率。
  • 使用 module 可支持 babel 这种预编译工具（在 webpack 里做为 loader 使用）。
  • 使用 eval 方式可大幅提高持续构建效率
  • eval-source-map 使用 DataUrl，本身包含完整 sourcemap 信息，并不像 sourceURL 那样，还需要浏览器发送一个完整请求去获取 sourcemap 文件，这会略微提高点效率

这中模式作为默认值，肯定有千好万好，但我在浏览器调试的时候，很多时候并不能准确定位到想要打断点的那一行，这就有点欺负人了，所以我将这个值换成了eval-source-map
